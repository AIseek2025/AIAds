# 仓库体积与 Git 忽略规则说明

## 背景

部署产物、归档包体积可达数 GB，若误加入 Git 历史，会导致：

- 仓库克隆与拉取极慢  
- GitHub 存储与 LFS 成本上升  
- 历史难以清理（大文件一旦提交会长期留在对象库中）

## 已配置的忽略项

在仓库根目录的 `.gitignore` 中已忽略：

- `*.zip`
- `*.tar.gz`
- `*.tgz`

请勿将上述类型的本地打包文件执行 `git add` 或强行 `git add -f` 提交。

## 本地文件 `aiads-deploy.zip` 等

若工作区仍存在例如 `aiads-deploy.zip` 等大型部署包：

- **仅保留在本地磁盘**即可，用于部署或备份  
- **不要**纳入版本控制  
- 若曾误提交，需使用 `git filter-repo` 或 BFG 等工具从历史中移除（必要时咨询团队规范）

## 建议实践

1. 大资产使用对象存储或制品库，不在 Git 中存放二进制大包。  
2. 发布前执行 `git status`，确认无意外的大文件。  
3. 持续集成里可对推送体积做检查（如 pre-push hook）。
