## 1. 广告主管理模块

### 1.1 模块概述

广告主管理模块是管理后台的核心功能之一，为运营人员和财务人员提供广告主全生命周期管理能力，包括广告主信息管理、认证审核、资金管理等。

**核心价值**:
- **高效管理**: 一站式管理所有广告主信息和状态
- **风险管控**: 严格审核广告主资质，降低平台风险
- **资金透明**: 完整的充值/消费记录，资金流向可追溯
- **运营支持**: 快速定位问题广告主，提升运营效率

### 1.2 功能清单

| 功能编号 | 功能名称 | 优先级 | 功能描述 | 目标用户 |
|---------|---------|-------|---------|---------|
| **ADV-001** | 广告主列表 | P0 | 分页展示广告主信息，支持搜索/筛选/排序 | 运营管理员、财务管理员 |
| **ADV-002** | 广告主详情 | P0 | 查看广告主完整信息、账户数据、活动统计 | 运营管理员、财务管理员 |
| **ADV-003** | 广告主审核 | P0 | 审核广告主企业认证资料 | 运营管理员 |
| **ADV-004** | 充值记录 | P0 | 查看广告主充值历史，支持导出 | 财务管理员、运营管理员 |
| **ADV-005** | 消费记录 | P0 | 查看广告主消费历史，支持导出 | 财务管理员、运营管理员 |
| **ADV-006** | 余额管理 | P0 | 调整广告主余额，冻结/解冻账户 | 超级管理员、财务管理员 |

### 1.5 操作流程

#### 1.5.1 广告主审核流程

```
1. 广告主提交企业认证申请
        ↓
2. 系统自动初审 (资料完整性检查)
        ↓
3. 运营管理员复审
        ↓
4. 审核通过 → 激活广告主账号 → 通知广告主
   审核拒绝 → 退回申请 → 通知广告主 (附原因)
```

#### 1.5.2 充值流程

```
1. 广告主提交充值申请
        ↓
2. 选择充值金额和支付方式
        ↓
3. 跳转支付平台完成支付
        ↓
4. 系统确认到账
        ↓
5. 更新账户余额 → 记录交易流水 → 通知广告主
```

#### 1.5.3 余额调整流程

```
1. 管理员发起余额调整申请
        ↓
2. 填写调整金额和原因
        ↓
3. 超级管理员审批 (大额调整需要)
        ↓
4. 执行调整 → 更新余额 → 记录审计日志 → 通知广告主
```

#### 1.5.4 冻结/解冻流程

```
冻结流程:
1. 管理员发起冻结申请
        ↓
2. 填写冻结原因
        ↓
3. 确认冻结 → 限制广告主操作 → 通知广告主

解冻流程:
1. 管理员发起解冻申请
        ↓
2. 填写解冻原因
        ↓
3. 确认解冻 → 恢复广告主权限 → 通知广告主
```

## 2. 活动管理模块

### 2.1 模块概述

活动管理模块为运营人员提供完整的活动生命周期管理能力，包括活动创建、审核、执行监控、效果分析等功能。

**核心价值**:
- **全流程管理**: 从创建到完成的全流程跟踪
- **智能审核**: AI 辅助审核，提高审核效率
- **数据驱动**: 实时效果监控，支持科学决策
- **风险预警**: 异常活动自动预警，降低风险

### 2.2 功能清单

| 功能编号 | 功能名称 | 优先级 | 功能描述 | 目标用户 |
|---------|---------|-------|---------|---------|
| **CMP-001** | 活动列表 | P0 | 分页展示活动信息，支持搜索/筛选/排序 | 所有管理员 |

### 2.5 操作流程

#### 2.5.1 活动创建流程

```
1. 广告主创建活动
        ↓
2. 填写活动信息和预算
        ↓
3. 设置 KOL 要求和目标受众
        ↓
4. 上传活动素材
        ↓
5. 提交审核 → 状态变为 pending_review
```

#### 2.5.2 活动审核流程

```
1. 系统自动初审 (内容合规性检查)
        ↓
2. 运营管理员复审
        ↓
3. 审核通过 → 状态变为 active → 开始匹配 KOL
   审核拒绝 → 状态变为 rejected → 通知广告主
```

#### 2.5.3 活动执行流程

```
1. 活动启动
        ↓
2. AI 匹配推荐 KOL
        ↓
3. KOL 接单创作
        ↓
4. 提交内容审核
        ↓
5. 审核通过 → 发布内容
        ↓
6. 数据追踪和监控
        ↓
7. 活动结束 → 状态变为 completed
```

#### 2.5.4 异常活动处理流程

```
1. 系统检测异常 (AI 模型/规则引擎)
        ↓
2. 生成异常告警
        ↓
3. 运营人员介入调查
        ↓
4. 确认异常 → 处理 (下架/处罚/赔偿)
   误报 → 标记为 false_positive
        ↓
5. 记录处理结果 → 优化检测模型
```

### 3.5 操作流程

#### 3.5.1 订单创建流程

```
1. 广告主选择 KOL 发起合作
        ↓
2. 填写合作要求和预算
        ↓
3. 确认订单信息并支付
        ↓
4. 订单创建 → 状态 pending_payment → pending_acceptance
        ↓
5. 通知 KOL 接单
```

#### 3.5.2 订单执行流程

```
1. KOL 接单 → 状态 in_progress
        ↓
2. KOL 创作内容
        ↓
3. 提交内容审核 → 状态 pending_review
        ↓
4. 广告主审核
        ↓
5. 审核通过 → 状态 pending_publish → 发布 → completed
   审核不通过 → 状态 pending_revision → 修改 → 重新审核
```

#### 3.5.3 订单纠纷处理流程

```
1. 任一方发起纠纷
        ↓
2. 填写纠纷原因和证据
        ↓
3. 系统通知对方回应
        ↓
4. 双方协商 (3 天内)
        ↓
5. 协商成功 → 纠纷关闭
   协商失败 → 申请平台仲裁
        ↓
6. 平台仲裁 (5 天内)
        ↓
7. 仲裁结果 → 执行 → 纠纷关闭
```

#### 3.5.4 订单取消流程

```
1. 任一方发起取消申请
        ↓
2. 填写取消原因
        ↓
3. 对方确认
        ↓
4. 确认取消 → 退款处理 → 订单关闭
   对方拒绝 → 进入纠纷流程
```

### 4.5 操作流程

#### 4.5.1 系统配置流程

```
1. 超级管理员进入系统配置页
        ↓
2. 修改配置参数
        ↓
3. 测试配置 (发送邮件/短信测试)
        ↓
4. 保存配置 → 验证配置有效性
        ↓
5. 配置生效 → 记录审计日志
```

#### 4.5.2 管理员创建流程

```
1. 超级管理员进入管理员管理页
        ↓
2. 点击创建管理员
        ↓
3. 填写管理员信息
        ↓
4. 选择角色和权限
        ↓
5. 设置初始密码
        ↓
6. 创建成功 → 发送通知邮件 → 记录审计日志
```

#### 4.5.3 角色权限配置流程

```
1. 超级管理员进入角色权限页
        ↓
2. 选择要编辑的角色
        ↓
3. 勾选/取消勾选权限
        ↓
4. 保存配置 → 验证权限配置
        ↓
5. 配置生效 → 通知该角色下管理员 → 记录审计日志
```

#### 4.5.4 敏感词管理流程

```
1. 管理员进入敏感词管理页
        ↓
2. 添加敏感词
        ↓
3. 设置分类和级别
        ↓
4. 保存生效 → 同步到内容审核系统
```
