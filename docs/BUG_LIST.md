### BUG-001: MFA 启用流程接口参数类型错误

| 属性 | 值 |
|-----|------|
| **模块** | 管理员认证 |
| **严重程度** | 一般 (Minor) |
| **状态** | 进行中 (In Progress) |
| **优先级** | P1 |
| **发现日期** | 2026-03-25 |
| **发现人** | QA Engineer |
| **负责人** | 后端开发团队 |

#### 问题描述
在启用 MFA 功能时，`enableMfa` 接口接收的参数类型与实际传递的参数不匹配，导致 TypeScript 编译错误。

#### 复现步骤
1. 管理员登录
2. 访问 MFA 设置页面
3. 点击启用 MFA
4. 输入验证码并提交

#### 预期结果
MFA 成功启用

#### 实际结果
TypeScript 编译错误：`Argument of type 'MfaResponse' is not assignable to parameter of type 'string'`

#### 技术细节
- **文件**: `/src/backend/src/controllers/admin/auth.controller.ts:181`
- **错误代码**: `const result = await adminAuthService.enableMfa(adminId, mfaSecret, req.body.mfaCode);`
- **问题**: `mfaSecret` 是对象，但 `enableMfa` 期望字符串

#### 修复建议
提取 `mfaSecret.secret` 传递给 `enableMfa` 方法。

- [x] 根因分析
