## 1. 系统介绍

### 1.1 产品定位

AIAds 管理后台是 AIAds 影响者营销平台的**运营管理中枢**，为平台运营人员、审核人员、财务人员和管理员提供统一的数字化运营管理界面。

**系统访问地址**: https://admin.aiads.com

### 1.2 目标用户

| 用户角色 | 描述 | 适用人员 |
|---------|------|---------|
| **超级管理员** | 系统最高权限，可管理所有功能和用户 | 技术负责人、CTO |
| **运营管理员** | 日常运营管理人员，负责用户管理、活动审核等 | 运营主管 |
| **审核管理员** | 内容/KOL 审核人员 | 审核团队 |
| **财务管理员** | 财务管理人员，负责充值提现审核 | 财务人员 |

### 1.3 核心价值

| 价值维度 | 具体描述 | 量化指标 |
|---------|---------|---------|
| **高效运营** | 一站式管理用户、KOL、内容、订单和财务 | 操作时间减少 50%+ |
| **智能审核** | AI 辅助审核，批量操作 | 审核效率提升 70%+ |
| **数据驱动** | 实时数据看板，支持科学决策 | 数据获取时间减少 80% |
| **安全可控** | 完善的权限体系和审计日志 | 100% 操作可追溯 |

## 2. 功能模块

### 2.1 功能模块总览

```
AIAds 管理后台
├── 1. 数据总览 (Dashboard)
├── 2. 用户管理 (User Management)
├── 3. 广告主管理 (Advertiser Management)
├── 4. KOL 管理 (KOL Management)
├── 5. 活动管理 (Campaign Management)
├── 6. 订单管理 (Order Management)
├── 7. 财务管理 (Finance Management)
├── 8. 内容审核 (Content Moderation)
├── 9. 数据统计 (Analytics)
└── 10. 系统设置 (System Settings)
```

### 2.2 数据总览 (Dashboard)

**功能描述**: 展示平台核心运营数据和待办事项

**核心功能**:
- 8 个核心数据卡片（总用户数、活跃广告主、认证 KOL、本月 GMV 等）
- 收入趋势图表（近 30 天）
- 用户增长趋势图表
- 最近活动列表
- 待办事项提醒

**权限要求**: `dashboard:view`（所有管理员）

**使用场景**:
- 每日上班第一件事查看平台运行状态
- 快速了解待处理任务
- 查看核心指标变化趋势

### 2.3 用户管理

**功能描述**: 管理平台所有用户账号

**核心功能**:
- 用户列表查看（支持分页、搜索、筛选）
- 用户详情查看（基本信息、行为数据）
- 用户编辑（修改用户信息）
- 用户封禁/解封
- 用户密码重置
- 用户删除（仅超级管理员）

**权限要求**:
- 查看：`user:view`
- 编辑：`user:edit`
- 封禁：`user:ban`
- 解封：`user:unban`
- 删除：`user:delete`（仅超级管理员）

**使用场景**:
- 处理用户违规行为
- 响应用户账号问题
- 管理用户状态

### 2.4 广告主管理

**功能描述**: 管理广告主账号和认证

**核心功能**:
- 广告主列表查看
- 广告主详情查看
- 企业认证审核
- 充值/消费记录查看
- 余额管理

**权限要求**:
- 查看：`advertiser:view`
- 审核：`advertiser:verify`
- 余额调整：`finance:adjustment`（仅超级管理员）

**使用场景**:
- 审核新广告主认证
- 查看广告主消费情况
- 处理广告主余额问题

### 2.5 KOL 管理

**功能描述**: 管理 KOL 账号和认证审核

**核心功能**:
- KOL 列表查看（支持多条件筛选）
- KOL 详情查看（基本信息、粉丝数据、互动率等）
- KOL 认证审核（通过/拒绝）
- 数据同步（手动同步社交平台数据）
- 评级管理（S/A/B/C）
- 黑名单管理

**权限要求**:
- 查看：`kol:view`
- 审核：`kol:review`, `kol:approve`, `kol:reject`
- 评级：`kol:verify`
- 黑名单：`kol:suspend`

**KOL 评级标准**:
| 评级 | 粉丝数 | 互动率 | 内容质量 | 合作履约率 |
|-----|-------|-------|---------|-----------|
| **S** | 5 万 + | 5%+ | 优秀 | 95%+ |
| **A** | 1 万 -5 万 | 3-5% | 良好 | 90%+ |
| **B** | 5 千 -1 万 | 2-3% | 一般 | 80%+ |
| **C** | 1 千 -5 千 | 1-2% | 合格 | 70%+ |

**使用场景**:
- 审核新 KOL 认证申请
- 定期同步 KOL 数据
- 调整 KOL 评级
- 处理违规 KOL

### 2.6 活动管理

**功能描述**: 管理平台营销活动

**核心功能**:
- 活动列表查看
- 活动详情查看
- 活动审核（通过/拒绝）
- 状态管理（上线/下线/暂停）
- 数据统计
- 异常监控

**权限要求**:
- 查看：`campaign:view`
- 审核：`campaign:review`
- 状态管理：`campaign:manage`

**活动状态流转**:
```
草稿 → 待审核 → 已审核 → 进行中 → 已完成
              ↓          ↓
            已拒绝    已暂停
```

**使用场景**:
- 审核新创建的活动
- 监控活动运行状态
- 处理异常活动

### 2.7 订单管理

**功能描述**: 管理广告订单

**核心功能**:
- 订单列表查看
- 订单详情查看
- 订单状态管理
- 纠纷处理
- 订单导出

**权限要求**:
- 查看：`order:view`
- 管理：`order:manage`
- 导出：`order:export`

**订单状态**:
| 状态 | 描述 | 可操作 |
|-----|------|-------|
| **待付款** | 广告主未付款 | 取消订单 |
| **待接单** | KOL 未接单 | 催促接单 |
| **进行中** | KOL 已接单，创作中 | - |
| **待审核** | KOL 提交内容，待审核 | 审核通过/拒绝 |
| **待发布** | 内容已审核，待发布 | - |
| **已完成** | 订单完成 | 评价 |
| **已取消** | 订单已取消 | - |
| **纠纷中** | 存在纠纷 | 介入处理 |

**使用场景**:
- 跟踪订单进度
- 处理订单纠纷
- 导出订单数据

### 2.8 财务管理

**功能描述**: 管理平台和用户财务

**核心功能**:
- 财务总览（核心财务指标）
- 充值管理（查看和管理充值记录）
- 提现审核（审核 KOL 提现申请）
- 交易记录（查看所有交易流水）
- 发票管理（管理发票申请和开具）
- 财务报表（生成和导出财务报表）

**权限要求**:
- 查看：`finance:view`
- 导出：`finance:export`
- 提现审核：`withdrawal:review`, `withdrawal:approve`, `withdrawal:reject`
- 余额调整：`finance:adjustment`（仅超级管理员）

**提现审核流程**:
```
1. KOL 提交提现申请
        ↓
2. 系统自动初审（余额检查、风控检查）
        ↓
3. 财务管理员复审
        ↓
4. 审核通过 → 打款 → 完成
   审核拒绝 → 退回 → 通知 KOL
```

**使用场景**:
- 审核提现申请
- 查看交易流水
- 生成财务报表

**使用场景**:

## 3. 操作流程

### 3.1 登录流程

```
1. 访问 https://admin.aiads.com
        ↓
2. 输入邮箱和密码
        ↓
3. 点击"登录"按钮
        ↓
4. （可选）输入 MFA 验证码
        ↓
5. 登录成功，跳转到数据总览
```

**注意事项**:
- 首次登录需要修改初始密码
- 建议启用 MFA 提高安全性
- 密码要求：至少 8 位，包含大小写字母、数字和特殊字符

### 3.2 KOL 审核流程

```
1. 进入"KOL 管理" → "待审核"
        ↓
2. 查看 KOL 提交的认证资料
        ↓
3. 核实 KOL 信息（粉丝数、互动率等）
        ↓
4. 点击"通过"或"拒绝"
        ↓
5. 填写审核意见（拒绝时必填）
        ↓
6. 提交审核结果
```

**注意事项**:
- 审核前需核实 KOL 社交平台数据
- 拒绝时需填写具体原因
- 支持批量审核

### 3.3 内容审核流程

```
1. 进入"内容审核" → "待审核"
        ↓
2. 查看内容详情（文字、图片、视频）
        ↓
3. 检查内容合规性
        ↓
4. 点击"通过"或"拒绝"
        ↓
5. 填写审核意见（拒绝时必填）
        ↓
6. 提交审核结果
```

**注意事项**:
- 审核时效要求：2 小时内完成
- 敏感内容需升级处理
- 支持批量审核

### 3.4 提现审核流程

```
1. 进入"财务管理" → "提现审核"
        ↓
2. 查看提现申请详情
        ↓
3. 核实 KOL 账户余额和交易记录
        ↓
4. 点击"通过"或"拒绝"
        ↓
5. 填写审核意见（拒绝时必填）
        ↓
6. 提交审核结果
        ↓
7. （通过后）财务打款并确认
```

**注意事项**:
- 大额提现（>¥10,000）需要二次验证
- 审核时效要求：24 小时内完成
- 支持批量审核

### 3.5 用户封禁流程

```
1. 进入"用户管理" → 找到目标用户
        ↓
2. 查看用户详情和违规行为
        ↓
3. 点击"封禁"按钮
        ↓
4. 选择封禁原因和封禁时长
        ↓
5. 填写封禁说明
        ↓
6. 确认封禁（需要二次验证）
        ↓
7. 封禁完成，用户无法登录
```

**注意事项**:
- 封禁操作需要二次验证
- 封禁记录记入审计日志
- 支持临时封禁和永久封禁

## 4. 常见问题

### 4.1 登录问题

**Q1: 忘记密码怎么办？**
- A: 点击登录页面的"忘记密码"链接，输入注册邮箱，系统将发送重置密码邮件。

**Q2: MFA 验证码收不到怎么办？**
- A: 
  1. 检查手机信号和网络
  2. 检查短信是否被拦截
  3. 联系超级管理员暂时关闭 MFA

**Q3: 账号被禁用怎么办？**
- A: 联系超级管理员或运营主管，说明情况申请解封。

### 4.2 权限问题

**Q1: 为什么看不到某个功能菜单？**
- A: 您的角色可能没有该功能的访问权限，请联系超级管理员配置权限。

**Q2: 为什么操作按钮是灰色的？**
- A: 您有查看权限但没有操作权限，需要申请相应权限。

**Q3: 如何申请更多权限？**
- A: 向直属主管申请，由超级管理员配置。

### 4.3 操作问题

**Q1: 批量审核如何操作？**
- A: 
  1. 在列表中勾选多个项目
  2. 点击顶部"批量审核"按钮
  3. 选择审核结果（通过/拒绝）
  4. 填写审核意见
  5. 确认提交

**Q2: 数据导出失败怎么办？**
- A: 
  1. 检查数据量是否过大（建议单次不超过 10 万条）
  2. 尝试缩小时间范围
  3. 如仍失败，联系技术支持

**Q3: 审核误操作怎么办？**
- A: 
  1. 立即联系超级管理员
  2. 查看审计日志确认操作记录
  3. 根据情况进行修正

### 4.4 系统问题

**Q1: 页面加载很慢怎么办？**
- A: 
  1. 检查网络连接
  2. 清除浏览器缓存
  3. 如持续缓慢，联系技术支持

**Q2: 系统提示错误怎么办？**
- A: 
  1. 记录错误信息和时间
  2. 刷新页面重试
  3. 如持续报错，联系技术支持

**Q3: 数据不一致怎么办？**
- A: 
  1. 刷新页面
  2. 清除缓存后重试
  3. 如仍不一致，联系技术支持排查
