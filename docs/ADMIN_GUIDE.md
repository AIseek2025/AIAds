## 1. 权限管理

### 1.1 权限体系概述

AIAds 管理后台采用 **RBAC（Role-Based Access Control）** 基于角色的访问控制模型。

```
用户 → 角色 → 权限 → 资源
```

**核心概念**:
- **用户**: 系统管理员账号
- **角色**: 权限的集合（如超级管理员、运营管理员等）
- **权限**: 对特定资源的操作能力（如 user:view, user:edit 等）
- **资源**: 系统功能模块（用户管理、KOL 管理等）

### 1.2 预定义角色

系统预定义 5 个角色，不可删除但可修改权限：

| 角色 | 英文名称 | 说明 | 适用人员 |
|-----|---------|------|---------|
| **超级管理员** | Super Admin | 系统最高权限，可管理所有功能和用户 | 技术负责人、CTO |
| **管理员** | Admin | 日常运营管理，用户管理，内容审核 | 运营主管 |
| **审核员** | Moderator | KOL 审核、内容审核 | 审核团队 |
| **财务** | Finance | 财务管理、提现审核 | 财务人员 |
| **数据分析师** | Analyst | 数据查看、报表导出 | 数据分析团队 |

### 1.4 自定义角色

**创建自定义角色**:
1. 进入"系统设置" → "角色权限"
2. 点击"创建角色"
3. 输入角色名称和描述
4. 勾选需要的权限
5. 点击"保存"

**修改角色权限**:
1. 在角色列表找到目标角色
2. 点击"编辑权限"
3. 调整权限勾选
4. 点击"保存"

**删除自定义角色**:
1. 在角色列表找到目标角色
2. 点击"删除"
3. 确认删除（系统预定义角色不可删除）

### 1.5 敏感操作二次验证

以下操作需要进行二次验证（MFA/短信/邮箱）:

| 操作类型 | 权限代码 | 验证方式 | 说明 |
|---------|---------|---------|------|
| **删除用户** | `user:delete` | MFA/短信 | 永久删除用户账号 |
| **余额调整** | `finance:adjustment` | MFA/短信 | 调整用户账户余额 |
| **大额提现审批** | `withdrawal:approve` | MFA | 单笔超过¥10,000 |
| **修改系统配置** | `settings:edit` | MFA | 修改核心系统参数 |
| **管理管理员** | `settings:admins:manage` | MFA | 创建/删除管理员 |
| **导出审计日志** | `settings:audit:export` | MFA | 导出敏感操作日志 |

## 2. 角色配置

### 2.1 查看角色列表

**操作步骤**:
1. 进入"系统设置" → "角色权限"
2. 查看角色列表

**角色列表信息**:
- 角色名称
- 角色描述
- 权限数量
- 关联管理员数量
- 是否系统预定义
- 操作（编辑/删除）

### 2.2 创建角色

**操作步骤**:
1. 进入"系统设置" → "角色权限"
2. 点击"创建角色"按钮
3. 填写角色信息：
   - 角色名称（必填）
   - 角色描述（必填）
   - 权限列表（勾选）
4. 点击"保存"

**角色命名规范**:
- 使用中文名称，便于理解
- 名称应反映角色职责
- 避免使用过于宽泛的名称

### 2.3 编辑角色

**修改角色信息**:
1. 在角色列表点击"编辑"
2. 修改角色名称或描述
3. 点击"保存"

**修改角色权限**:
1. 在角色列表点击"编辑权限"
2. 勾选/取消勾选权限
3. 点击"保存"

**权限分组**:
- 数据总览权限
- 用户管理权限
- 广告主管理权限
- KOL 管理权限
- 活动管理权限
- 订单管理权限
- 财务管理权限
- 内容审核权限
- 数据统计权限
- 系统设置权限

### 2.4 删除角色

**删除条件**:
- 仅自定义角色可删除
- 系统预定义角色不可删除
- 角色下无关联管理员

**删除步骤**:
1. 在角色列表点击"删除"
2. 确认删除
3. 如有关联管理员，需先转移管理员到其他角色

### 2.5 角色分配

**给管理员分配角色**:
1. 进入"系统设置" → "管理员管理"
2. 找到目标管理员
3. 点击"编辑"
4. 选择角色
5. 点击"保存"

**一个管理员只能有一个角色**，如需多个角色的权限，需创建包含所有权限的新角色。

## 3. 审计日志查看

### 3.1 审计日志概述

审计日志记录所有管理员的操作行为，用于：
- 操作追溯
- 安全审计
- 问题排查
- 合规要求

**日志保留期限**: 180 天

### 3.2 查看审计日志

**操作步骤**:
1. 进入"系统设置" → "操作日志"
2. 查看日志列表

**日志列表字段**:
- 操作时间
- 操作人（管理员）
- 操作类型
- 操作模块
- 操作对象
- 操作结果（成功/失败）
- IP 地址
- 操作详情

### 3.3 搜索审计日志

**搜索条件**:
- 时间范围：自定义起止时间
- 操作人：选择或搜索管理员
- 操作类型：查看/创建/编辑/删除/审核等
- 操作模块：用户管理/KOL 管理/财务管理等
- 操作结果：全部/成功/失败
- IP 地址：精确匹配

**组合搜索**:
- 支持多个条件组合搜索
- 搜索条件可保存为常用搜索

### 3.4 导出审计日志

**导出步骤**:
1. 设置搜索条件筛选日志
2. 点击"导出"按钮
3. 选择导出格式（Excel/CSV）
4. 进行二次验证（MFA）
5. 下载导出文件

**导出限制**:
- 单次导出最多 10 万条记录
- 导出文件包含敏感信息，需妥善保管
- 导出操作本身会被记录到审计日志

### 3.5 常见审计场景

#### 查看用户封禁记录

**搜索条件**:
- 操作类型：封禁
- 操作模块：用户管理
- 时间范围：自定义

#### 查看财务操作记录

**搜索条件**:
- 操作类型：审核/调整
- 操作模块：财务管理
- 时间范围：自定义

#### 查看登录记录

**搜索条件**:
- 操作类型：登录
- 操作模块：认证
- 时间范围：自定义

#### 查看配置变更记录

**搜索条件**:
- 操作类型：编辑
- 操作模块：系统设置
- 时间范围：自定义

## 5. 管理员管理

### 5.1 管理员列表

**查看管理员列表**:
1. 进入"系统设置" → "管理员管理"
2. 查看管理员列表

**列表字段**:
- 管理员 ID
- 姓名
- 邮箱
- 角色
- 状态（启用/禁用）
- 创建时间
- 最后登录时间
- 操作（编辑/禁用/删除）

### 5.2 创建管理员

**操作步骤**:
1. 进入"系统设置" → "管理员管理"
2. 点击"创建管理员"按钮
3. 填写管理员信息：
   - 姓名（必填）
   - 邮箱（必填，作为登录账号）
   - 角色（必填）
   - 初始密码（必填）
   - 是否强制修改密码（可选）
   - 是否启用 MFA（可选）
4. 进行二次验证（MFA）
5. 点击"保存"

**初始密码处理**:
- 建议勾选"强制修改密码"
- 新管理员首次登录需修改密码
- 初始密码通过安全渠道告知管理员

### 5.3 编辑管理员

**修改基本信息**:
1. 在管理员列表点击"编辑"
2. 修改姓名等信息
3. 点击"保存"

**修改角色**:
1. 在管理员列表点击"编辑"
2. 选择新角色
3. 点击"保存"
4. 角色变更立即生效

**重置密码**:
1. 在管理员列表点击"重置密码"
2. 输入新密码
3. 进行二次验证（MFA）
4. 点击"确认"
5. 将新密码告知管理员

### 5.4 禁用/启用管理员

**禁用管理员**:
1. 在管理员列表点击"禁用"
2. 填写禁用原因
3. 进行二次验证（MFA）
4. 确认禁用

**禁用效果**:
- 管理员无法登录
- 现有 Session 失效
- 保留管理员记录

**启用管理员**:
1. 在管理员列表找到已禁用的管理员
2. 点击"启用"
3. 确认启用

### 5.5 删除管理员

**删除条件**:
- 管理员已禁用
- 管理员无进行中的工作

**删除步骤**:
1. 在管理员列表点击"删除"
2. 确认删除
3. 进行二次验证（MFA）

**删除效果**:
- 管理员账号永久删除
- 操作记录保留在审计日志

### 5.6 管理员最佳实践

#### 账号安全

- 为新管理员分配最小必要权限
- 强制要求启用 MFA
- 定期审查管理员权限
- 及时禁用离职管理员账号

#### 权限管理

- 遵循最小权限原则
- 定期审查角色权限
- 敏感权限谨慎分配
- 权限变更需审批

## 7. 备份管理

### 7.1 备份策略

#### 数据库备份

| 备份类型 | 频率 | 保留期限 |
|---------|------|---------|
| 全量备份 | 每天 02:00 | 30 天 |
| 增量备份 | 每小时 | 7 天 |
| 手动备份 | 按需 | 永久 |

#### 配置备份

| 备份项 | 频率 | 存储位置 |
|-------|------|---------|
| 环境变量 | 每次变更 | Railway + 本地 |
| 系统配置 | 每次变更 | 数据库 + 本地 |
| 代码版本 | 每次部署 | Git + Railway |

### 7.2 手动备份

#### 数据库手动备份

**操作步骤**:
1. 进入"系统设置" → "备份管理"
2. 点击"创建备份"按钮
3. 选择备份类型（全量/增量）
4. 填写备份说明
5. 点击"确认"

**备份文件**:
- 格式：SQL 压缩文件
- 命名：`backup_YYYYMMDD_HHMMSS.sql.gz`
- 存储：Cloudflare R2 + 本地

#### 配置手动备份

**操作步骤**:
1. 进入"系统设置" → "备份管理"
2. 点击"备份配置"按钮
3. 选择备份项
4. 点击"确认"

### 7.3 恢复操作

#### 数据库恢复

**恢复步骤**:
1. 进入"系统设置" → "备份管理"
2. 点击"恢复"标签
3. 选择备份文件
4. 确认恢复（需要二次验证）
5. 等待恢复完成

**注意事项**:
- 恢复操作会覆盖当前数据
- 恢复前建议先备份当前数据
- 恢复操作需在低峰期进行
- 恢复后需验证数据完整性

#### 配置恢复

**恢复步骤**:
1. 进入"系统设置" → "备份管理"
2. 点击"恢复"标签
3. 选择配置备份
4. 选择要恢复的配置项
5. 确认恢复

### 7.4 备份验证

#### 定期恢复测试

**测试频率**: 每月一次

**测试步骤**:
1. 选择最近的备份文件
2. 在测试环境恢复
3. 验证数据完整性
4. 记录测试结果

#### 备份完整性检查

**检查项**:
- 备份文件大小是否正常
- 备份文件是否可解压
- SQL 文件是否可执行
- 数据量是否匹配
